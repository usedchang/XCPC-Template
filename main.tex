\subsection{根号分治}


\begin{itemize}
    \item 第一行：两个整数 $n, q$（$1 \le n, q \le 10^5$），分别表示序列长度和询问次数。
    \item 第二行：$n$ 个整数 $A_1, A_2, \dots, A_n$（$1 \le A_i \le 10^9$），表示给定的整数序列。
    \item 接下来 $q$ 行：每行两个整数 $x, y$（$1 \le x, y \le 10^9$），表示一次询问。
\end{itemize}

\section*{输出格式}
输出 $q$ 行，每行一个整数，表示对应询问的答案。

\section*{简要分析}
给定序列 $A[1..n]$ 和询问 $(x,y)$，需统计满足 $i<j$ 且 $A[i]=x, A[j]=y$ 的数对数量。记 $pos[v]$ 为值 $v$ 在序列中出现的位置集合（升序排列），则答案可表示为：
$$
\sum_{p \in pos[x]} \left( |pos[y]| - \text{upper\_bound}(pos[y], p) \right)
$$
或等价表示为：
$$
\sum_{p \in pos[y]} \text{lower\_bound}(pos[x], p)
$$

\section*{核心划分}
设阈值 $B = \sqrt{n}$（取整），将所有值分为两类：
\begin{enumerate}
    \item \textbf{高频值（Heavy）}：满足 $|pos[v]| > B$ 的值，记高频值集合为 $H$；
    \item \textbf{低频值（Light）}：满足 $|pos[v]| \le B$ 的值。
\end{enumerate}

\section*{预处理阶段（离线预处理高频值对）}
对于任意两个高频值 $u, v \in H$，预计算数对 $(u,v)$ 的答案并存储在哈希表 $mp$ 中：
$$
mp[(u,v)] = \sum_{p \in pos[u]} \left( |pos[v]| - \text{upper\_bound}(pos[v], p) \right)
$$
预处理时间复杂度：$O(|H|^2 \cdot B)$，由于 $|H| \le \frac{n}{B}$，故总复杂度为 $O(nB)$。

\section*{查询阶段（分情况处理）}
对每个询问 $(x,y)$，分三类处理（结合记忆化避免重复计算）：
\begin{enumerate}
    \item 若 $(x,y)$ 已计算过（记忆化哈希表 $\text{memo}$ 存在），直接返回结果；
    \item 若 $x,y$ 均为高频值，直接调用预处理结果 $mp[(x,y)]$；
    \item 若至少一个为低频值，选择更小的集合遍历计算：
          \begin{itemize}
              \item 若 $|pos[x]| \le |pos[y]|$：遍历 $pos[x]$，对每个位置 $p$ 统计 $pos[y]$ 中大于 $p$ 的元素数量；
              \item 否则：遍历 $pos[y]$，对每个位置 $p$ 统计 $pos[x]$ 中小于 $p$ 的元素数量。
          \end{itemize}
          单次查询时间复杂度：$O(B)$。
\end{enumerate}

\section*{复杂度分析}
预处理复杂度：$O(n\sqrt{n})$；
单次查询复杂度：$O(\sqrt{n})$，总查询复杂度：$O(q\sqrt{n})$；
整体时间复杂度：$O((n+q)\sqrt{n})$，可通过 $10^5$ 规模的数据。

\subsubsection{无向图三元环计数}
\section*{题面简述}

给定一个 $n$ 个点 $m$ 条边的简单无向图（无重边、无自环，不保证连通），求图中三元环的个数。

三元环指恰好包含三个点 $u, v, w$ 及三条边 $\langle u,v\rangle, \langle v,w\rangle, \langle w,u\rangle$ 的子图。两个三元环不同当且仅当存在一个点属于其中一个而不属于另一个。

数据范围：$1 \le n \le 10^5$，$1 \le m \le 2 \times 10^5$。

\section*{做法}

采用经典的\textbf{无向边定向 + 枚举}算法。

\begin{enumerate}
    \item \textbf{定向}：对每条无向边 $(u, v)$，令其从度数小的点指向度数大的点；若度数相同，则从编号小的点指向编号大的点。这样将无向图转化为一个 DAG，记有向边集为 $E'$。
    \item \textbf{枚举}：对每个点 $u$，先用标记数组标记 $u$ 的所有出邻居。然后对每条出边 $u \to v$，枚举 $v$ 的所有出邻居 $w$，若 $w$ 被 $u$ 标记过（即 $u \to w \in E'$），则 $(u, v, w)$ 构成一个三元环，答案加一。
\end{enumerate}

由于定向后每个三元环的三条边恰好形成一条有向路径 $u \to v \to w$（其中 $u \to w$ 也存在），因此每个三元环恰好被统计一次，无需去重。

\section*{复杂度分析}

\textbf{引理}：定向后每个点的出度 $d^+(u) \le \sqrt{2m}$。

\begin{proof}
对任意点 $u$，其每个出邻居 $v$ 都满足 $\deg(v) \ge \deg(u) \ge d^+(u)$。因此 $u$ 的 $d^+(u)$ 个出邻居的度数之和满足：
\[
    \sum_{v \in N^+(u)} \deg(v) \ge (d^+(u))^2
\]
又因为所有边的度数贡献总和不超过 $2m$，故 $(d^+(u))^2 \le 2m$，即 $d^+(u) \le \sqrt{2m}$。
\end{proof}

算法的核心操作量为：
\[
    T = \sum_{(u \to v) \in E'} d^+(v) \le \sum_{(u \to v) \in E'} \sqrt{2m} = |E'| \cdot \sqrt{2m} = m\sqrt{2m}
\]

因此总时间复杂度为 $O(m\sqrt{m})$。

对于本题 $m = 2 \times 10^5$，操作量约为 $2 \times 10^5 \times 632 \approx 1.26 \times 10^8$，可以通过。空间复杂度为 $O(n + m)$。

\subsection{树上启发式合并}
\section*{题面简述}

给一棵以 $1$ 为根的树，共 $n$ 个结点，每个结点有一个颜色 $c[i]$。每次询问给出一个结点 $u$，求以 $u$ 为根的子树中有多少种不同的颜色。

\subsection*{第一步：求 DFS 序与子树区间}

对树进行一次 DFS，记录每个结点的进入时间 $\mathrm{in}[u]$ 和离开时间 $\mathrm{out}[u]$。则以 $u$ 为根的子树恰好对应 DFS 序中的连续区间 $[\mathrm{in}[u],\;\mathrm{out}[u]]$。

同时在 DFS 过程中记录每个结点的\textbf{重儿子}（子树大小最大的儿子），记为 $\mathrm{heavy}[u]$。

\subsection*{第二步：启发式合并（DSU on Tree）}

对每个结点 $u$，按如下顺序处理：

\begin{enumerate}
    \item 递归处理所有\textbf{轻儿子}，处理完后\textbf{清除}其对全局计数数组的贡献。
    \item 递归处理\textbf{重儿子}，处理完后\textbf{保留}其对全局计数数组的贡献。
    \item 暴力遍历所有轻儿子的子树中的结点，将它们的颜色加入全局计数数组 $\mathrm{cnt}[\cdot]$：
    \[
        \mathrm{cnt}[c[v]] \leftarrow \mathrm{cnt}[c[v]] + 1
    \]
    若某颜色 $c[v]$ 的计数从 $0$ 变为 $1$，则当前不同颜色种类数加一：
    \[
        \mathrm{tot} \leftarrow \mathrm{tot} + [\mathrm{cnt}[c[v]] = 1]
    \]
    \item 将结点 $u$ 自身的颜色也加入计数数组，同理更新 $\mathrm{tot}$。
    \item 此时 $\mathrm{tot}$ 即为以 $u$ 为根的子树的颜色种类数，记录答案：
    \[
        \mathrm{ans}[u] = \mathrm{tot}
    \]
    \item 若 $u$ 本身是其父结点的轻儿子，则暴力清除整棵子树的贡献（将 $\mathrm{cnt}[\cdot]$ 和 $\mathrm{tot}$ 恢复）。
\end{enumerate}

\subsection*{第三步：回答询问}

所有询问离线处理。对于每个询问的结点 $u$，直接输出 $\mathrm{ans}[u]$ 即可。

\section*{正确性说明}

启发式合并的核心思想在于：每次只保留重儿子的信息，轻儿子的信息暴力重算。由于每个结点到根的路径上至多有 $O(\log n)$ 条轻边（每经过一条轻边，子树大小至少翻倍），所以每个结点被暴力遍历的总次数为 $O(\log n)$。

\section*{复杂度分析}

\begin{itemize}
    \item \textbf{时间复杂度：}每个结点被暴力加入/清除计数数组的次数为其到根路径上的轻边数，至多 $O(\log n)$ 次。因此总时间复杂度为：
    \[
        O(n \log n + m)
    \]
    其中 $m$ 为询问数。

\end{itemize}

\subsection{树上背包}

\textbf{25上海市赛I}

\section*{题目大意}
给定一棵以 $1$ 为根的有根树，每个节点上的人要么是“诚实者”（恒说真话），要么是“说谎者”（恒说假话）。每个节点 $i$ 的人声称：“以我为根的子树中，恰好有 $a_i$ 个诚实者。”
求所有满足以下逻辑自洽条件的身份分配方案数：
\begin{itemize}
    \item 若节点 $i$ 是诚实者，则其子树中诚实者的实际数量必须等于 $a_i$；
    \item 若节点 $i$ 是说谎者，则其子树中诚实者的实际数量必须不等于 $a_i$。
\end{itemize}

\begin{itemize}
    \item 第一行：一个整数 $T$（$1 \le T \le 5000$），表示数据组数。
    \item 每组数据第一行：一个正整数 $n$（$1 \le n \le 5000$，$\sum n \le 5000$），表示节点数量。
    \item 每组数据第二行：$n$ 个非负整数 $a_1, a_2, \dots, a_n$（$0 \le a_i \le n$），表示每个节点的陈述。
    \item 接下来 $n-1$ 行：每行两个正整数 $u_i, v_i$（$1 \le u_i, v_i \le n$），表示树的一条边，保证输入构成一棵以 $1$ 为根的有根树。
\end{itemize}

\section*{输出格式}
对每组数据输出一行一个整数，表示满足条件的真假分配情况数量，答案对 $998244353$ 取模。


\section*{简要分析}
采用树形背包 DP 求解，通过后序遍历合并子树状态，根据节点身份（诚实/说谎）约束转移，统计合法方案数。

\section*{状态定义}
\begin{itemize}
    \item $f[u][j]$：仅用节点 $u$ 的子树，产生 $j$ 个诚实者的合法方案数（不包含 $u$ 自身）；
    \item $dp[u][j]$：用节点 $u$ 的子树 + $u$ 自身，产生 $j$ 个诚实者的合法方案数；
    \item $siz[u]$：以 $u$ 为根的子树大小。
\end{itemize}

\section*{状态转移}
1. 初始化：$f[u][0] = 1$，$siz[u] = 1$；
2. 子树合并：遍历子节点 $v$，用**树形背包**倒序枚举合并状态：
   $$nxt[j+k] += f[u][j] \times dp[v][k] \pmod{mod}$$
   用 $nxt$ 更新 $f[u]$，并更新子树大小；
3. 身份转移：
   - 说谎者：$dp[u][j] = f[u][j] \ (j \neq a[u])$；
   - 诚实者：$dp[u][a[u]] = f[u][a[u]-1] \ (a[u] \ge 1)$。

\section*{复杂度分析}
代码虽为三重循环，本质是树形背包均摊复杂度：
任意两个节点仅在其最近公共祖先处被枚举一次，总操作数等于子树大小乘积和，**严格为 $O(n^2)$**；
结合题目约束 $\sum n \le 5000$，总计算量不超过 $2.5 \times 10^7$，可轻松通过。

\section*{答案统计}
最终答案为根节点 $1$ 的所有合法状态之和：$\sum\limits_{i=0}^{siz[1]} dp[1][i] \pmod{998244353}$。

\subsection{数论分块}
\section*{题面简述}

给定一个正整数 $x$，求 $\sum_{i=1}^{x} d(i)$，其中 $d(i)$ 表示 $i$ 的因数个数。


\subsection*{第一步：转化求和式}

由因数个数的定义 $d(i) = \sum_{d \mid i} 1$，将求和交换顺序：
\[
    \sum_{i=1}^{x} d(i) = \sum_{i=1}^{x} \sum_{d \mid i} 1 = \sum_{d=1}^{x} \left\lfloor \frac{x}{d} \right\rfloor
\]

即问题转化为求 $\sum_{d=1}^{x} \left\lfloor \frac{x}{d} \right\rfloor$。

\subsection*{第二步：整除分块}

$\left\lfloor \frac{x}{d} \right\rfloor$ 的取值至多只有 $O(\sqrt{x})$ 种。对于每一种取值 $v = \left\lfloor \frac{x}{l} \right\rfloor$，所有满足 $\left\lfloor \frac{x}{d} \right\rfloor = v$ 的 $d$ 构成一个连续区间 $[l, r]$，其中：
\[
    r = \min\!\left(\left\lfloor \frac{x}{v} \right\rfloor,\; x\right)
\]

在该区间内 $\left\lfloor \frac{x}{d} \right\rfloor$ 恒为 $v$，因此该块的贡献为：
\[
    v \times (r - l + 1)
\]

遍历所有块，累加贡献即得答案：
\[
    \mathrm{Ans} = \sum_{\text{block } [l,r]} \left\lfloor \frac{x}{l} \right\rfloor \times (r - l + 1)
\]

每次令 $l \leftarrow r + 1$ 跳到下一个块，直到 $l > x$。

\section*{复杂度分析}

$\left\lfloor \frac{x}{d} \right\rfloor$ 的不同取值个数为 $O(\sqrt{x})$（当 $d \le \sqrt{x}$ 时至多 $\sqrt{x}$ 种，当 $d > \sqrt{x}$ 时 $\left\lfloor \frac{x}{d} \right\rfloor < \sqrt{x}$，同样至多 $\sqrt{x}$ 种），因此整除分块的块数为 $O(\sqrt{x})$，每块内仅需 $O(1)$ 计算。

总时间复杂度为 $O(\sqrt{x})$，空间复杂度为 $O(1)$。

\subsection{莫比乌斯反演}
\section*{题面简述}

给定 $N, M$，求满足 $1 \le x \le N$，$1 \le y \le M$ 且 $\gcd(x, y)$ 为质数的有序对 $(x, y)$ 的个数。

多组询问，$T \le 10^4$，$N, M \le 10^7$。


\subsection*{第一步：枚举质数}

设答案为 $\mathrm{Ans}$，直接按定义展开：
\[
    \mathrm{Ans} = \sum_{p \in \mathcal{P}} \sum_{x=1}^{N} \sum_{y=1}^{M} [\gcd(x, y) = p]
\]
其中 $\mathcal{P}$ 为质数集合。令 $x = pa$，$y = pb$，则：
\[
    \mathrm{Ans} = \sum_{p \in \mathcal{P}} \sum_{a=1}^{\lfloor N/p \rfloor} \sum_{b=1}^{\lfloor M/p \rfloor} [\gcd(a, b) = 1]
\]

\subsection*{第二步：莫比乌斯反演}

利用经典恒等式 $[\gcd(a,b)=1] = \sum_{d \mid \gcd(a,b)} \mu(d)$，代入得：
\[
    \mathrm{Ans} = \sum_{p \in \mathcal{P}} \sum_{d=1}^{\min(\lfloor N/p \rfloor, \lfloor M/p \rfloor)} \mu(d) \left\lfloor \frac{N}{pd} \right\rfloor \left\lfloor \frac{M}{pd} \right\rfloor
\]

\subsection*{第三步：换元合并}

令 $T = pd$，则 $d = T/p$，对于固定的 $T$，$p$ 遍历 $T$ 的所有质因子。交换求和顺序：
\[
    \mathrm{Ans} = \sum_{T=1}^{\min(N,M)} \left\lfloor \frac{N}{T} \right\rfloor \left\lfloor \frac{M}{T} \right\rfloor \sum_{p \mid T,\, p \in \mathcal{P}} \mu\!\left(\frac{T}{p}\right)
\]

定义函数：
\[
    f(T) = \sum_{p \mid T,\, p \in \mathcal{P}} \mu\!\left(\frac{T}{p}\right)
\]

则答案化为：
\[
    \mathrm{Ans} = \sum_{T=1}^{\min(N,M)} f(T) \left\lfloor \frac{N}{T} \right\rfloor \left\lfloor \frac{M}{T} \right\rfloor
\]

\subsection*{第四步：预处理 $f$ 及其前缀和}

$f(T)$ 的预处理方式：线性筛求出所有质数与 $\mu$ 值后，枚举每个质数 $p$，再枚举 $p$ 的倍数 $T = kp$，将 $\mu(k)$ 累加到 $f(T)$ 上。这一步复杂度为：
\[
    \sum_{p \in \mathcal{P}} \left\lfloor \frac{n}{p} \right\rfloor \approx O\!\left(\frac{n \ln \ln n}{1}\right) = O(n \log \log n)
\]

随后求出 $f$ 的前缀和 $F(n) = \sum_{T=1}^{n} f(T)$。

\subsection*{第五步：整除分块回答询问}

对每组询问，利用 $\lfloor N/T \rfloor$ 和 $\lfloor M/T \rfloor$ 的取值分块性质（整除分块/数论分块），将 $T$ 的求和区间划分为 $O(\sqrt{N} + \sqrt{M})$ 个块，每块内 $\lfloor N/T \rfloor$ 和 $\lfloor M/T \rfloor$ 均为常数，利用前缀和 $F$ 即可 $O(1)$ 求出每块的贡献：
\[
    \mathrm{Ans} = \sum_{\text{block } [l,r]} \left\lfloor \frac{N}{l} \right\rfloor \left\lfloor \frac{M}{l} \right\rfloor \bigl(F(r) - F(l-1)\bigr)
\]

\section*{复杂度分析}

\begin{itemize}
    \item \textbf{预处理}：线性筛求 $\mu$ 和质数表为 $O(n)$；枚举质数倍数计算 $f$ 为 $O(n \log \log n)$；前缀和为 $O(n)$。总预处理复杂度 $O(n \log \log n)$，其中 $n = \max(N, M) = 10^7$。
    \item \textbf{单次询问}：整除分块为 $O(\sqrt{n})$。
    \item \textbf{总复杂度}：$O(n \log \log n + T\sqrt{n})$。
\end{itemize}

对于本题 $n = 10^7$，$T = 10^4$：
\begin{itemize}
    \item 预处理：$10^7 \times \log \log(10^7) \approx 10^7 \times 3 \approx 3 \times 10^7$。
    \item 查询：$10^4 \times \sqrt{10^7} \approx 10^4 \times 3162 \approx 3.16 \times 10^7$。
\end{itemize}

总操作量约 $6 \times 10^7$，可以通过。空间复杂度为 $O(n)$。

\subsection{多路归并(含二分解)}
\section*{题面简述}

给定两个长度为 $N$ 的单调不降序列 $A, B$，从 $A, B$ 中各取一个数相加可得到 $N^2$ 个和，求这 $N^2$ 个和中最小的 $N$ 个。

\subsection*{第一步：小根堆初始化与去重策略}

由于序列 $A, B$ 均单调不降，最小的和必然是 $A_1 + B_1$（下标从 1 开始）。我们使用一个小根堆（优先队列）维护候选和，堆中元素为三元组 $(sum, i, j)$，表示 $A_i + B_j$ 的值及其对应下标。

为避免重复访问同一 $(i, j)$ 对，采用如下策略：
\begin{itemize}
    \item 初始将 $(A_1 + B_1, 1, 1)$ 入堆，并标记 $(1, 1)$ 为已访问。
    \item 每次取出堆顶 $(sum, i, j)$ 加入答案后：
    \begin{enumerate}
        \item 若 $j + 1 \leq N$ 且 $(i, j+1)$ 未访问，将 $(A_i + B_{j+1}, i, j+1)$ 入堆并标记。
        \item 若 $j = 1$（当前是第 $i$ 行首个元素）且 $i + 1 \leq N$，将 $(A_{i+1} + B_j, i+1, j)$ 入堆并标记。
    \end{enumerate}
\end{itemize}

该策略确保每个 $(i, j)$ 仅入堆一次，避免重复计算。

\subsection*{第二步：提取前 $N$ 个最小和}

重复执行 $N$ 次以下操作：
\begin{enumerate}
    \item 取出堆顶元素，将其和加入答案数组。
    \item 按上述策略将后续候选和入堆。
\end{enumerate}

最终答案数组即为所求的 $N$ 个最小和。

\section*{正确性说明}

由于 $A, B$ 单调不降，对任意 $(i, j)$ 有 $A_i + B_j \leq A_{i+1} + B_j$ 且 $A_i + B_j \leq A_i + B_{j+1}$。小根堆保证每次取出当前最小候选和，去重策略确保无重复入堆，且不遗漏任何可能的最小和。


\lstinputlisting[caption={堆}]{cpp_templates/Other Trick/堆实现多路归并.cpp}

\subsection*{第一步：二分查找确定第 $N$ 小的和的值 $x$}

定义函数 $\mathrm{calc}(mid)$，用于计算序列 $A, B$ 中满足 $A_i + B_j \leq mid$ 的数对 $(i, j)$ 的数量。利用单调性，该函数可通过双指针法在 $O(N)$ 时间内实现：
\begin{itemize}
    \item 初始化指针 $j = N - 1$（指向 $B$ 的末尾）。
    \item 遍历 $A$ 的每个元素 $A_i$：
    \begin{itemize}
        \item 若 $A_i + B_j > mid$，则 $j$ 左移（减小 $j$）。
        \item 此时 $B_0$ 到 $B_j$ 与 $A_i$ 的和均 $\leq mid$，贡献 $j + 1$ 个计数。
    \end{itemize}
\end{itemize}

二分查找的初始区间设为 $[l, r]$，其中 $l = A_0 + B_0$（最小和），$r = A_{N-1} + B_{N-1}$（最大和）。二分过程如下：
\begin{enumerate}
    \item 取中点 $mid = l + \frac{r - l}{2}$。
    \item 若 $\mathrm{calc}(mid) \geq N$，说明第 $N$ 小的和 $\leq mid$，令 $r = mid$。
    \item 否则，说明第 $N$ 小的和 $> mid$，令 $l = mid + 1$。
    \item 当 $l = r$ 时，$x = l$ 即为第 $N$ 小的和的值。
\end{enumerate}

\subsection*{第二步：收集所有严格小于 $x$ 的和}

再次利用双指针法，遍历序列 $A$：
\begin{itemize}
    \item 对每个 $A_i$，找到最大的 $j$ 使得 $A_i + B_j < x$。
    \item 将 $A_i + B_0, A_i + B_1, \dots, A_i + B_j$ 全部加入结果数组 $\mathrm{res}$。
\end{itemize}

\subsection*{第三步：补充等于 $x$ 的和至 $N$ 个}

计算还需补充的数量 $\mathrm{need} = N - \mathrm{res.size()}$。若 $\mathrm{need} > 0$，再次遍历序列 $A$：
\begin{itemize}
    \item 对每个 $A_i$，找到最大的 $j$ 使得 $A_i + B_j \leq x$。
    \item 从 $j$ 开始向左遍历，若 $A_i + B_k == x$，则将 $x$ 加入 $\mathrm{res}$，并减少 $\mathrm{need}$，直到 $\mathrm{need} = 0$。
\end{itemize}

\subsection*{第四步：排序并输出}

对结果数组 $\mathrm{res}$ 进行排序，取前 $N$ 个元素依次输出，即为所求的 $N$ 个最小和。

\section*{正确性说明}

二分查找确保找到的 $x$ 是满足“和 $\leq x$ 的数对数量 $\geq N$”的最小值，因此 $x$ 必为第 $N$ 小的和。通过先收集所有小于 $x$ 的和，再补充等于 $x$ 的和，确保结果数组包含恰好 $N$ 个最小和，排序后顺序正确。

\section*{复杂度分析}

\begin{itemize}
    \item \textbf{时间复杂度：}
    \begin{itemize}
        \item 二分查找部分：共进行 $O(\log S)$ 次二分（$S$ 为和的范围，约 $2 \times 10^9$），每次 $\mathrm{calc}$ 函数耗时 $O(N)$，总耗时 $O(N \log S)$。
        \item 收集与补充和部分：两次双指针遍历均为 $O(N)$，最后排序为 $O(N \log N)$。
        \item 综上，总时间复杂度为：
        \[
            O(N \log N + N \log S)
        \]
    \end{itemize}
    \item \textbf{空间复杂度：} 结果数组 $\mathrm{res}$ 需 $O(N)$ 空间，总空间复杂度为 $O(N)$。
\end{itemize}

\subsection{Log Trick}
前缀 GCD ,OR , AND 等操作变化的级别是呈现对数级别的.
\section*{问题简述(Hdu 5869)}
给定一个包含 $N$ 个正整数 $a_1, a_2, \dots, a_{N-1}, a_N$ 的数组 $a$；定义 $a$ 的子数组为 $a_1$ 到 $a_N$ 之间的连续区间。换句话说，对于 $1 \leq i \leq j \leq N$ 而言，$a_i, a_{i+1}, \dots, a_{j-1}, a_j$ 就是 $a$ 的一个子数组。对于形如 $(L, R)$ 的查询，请回答区间 $[L, R]$ 内所有子数组产生的不同GCD值的数量。


数据保证：
\begin{itemize}
    \item $1 \leq N, Q \leq 100000$
    \item $1 \leq a_i \leq 1000000$
\end{itemize}

\subsection*{核心性质观察}
对于数组中的每个位置 $i$（作为子数组的右端点），所有以 $i$ 为右端点的子数组的 GCD 具有以下性质：
\begin{itemize}
    \item 这些 GCD 的数量是 $O(\log a_i)$ 级别的。因为每次将新的 $a_i$ 与前序 GCD 取新的 GCD 时，数值至少减半（或保持不变），因此不同 GCD 的数量随右端点递增呈对数级增长。
    \item 可以通过维护一个列表来记录这些 GCD 及其对应的最左左端点。例如，列表中的元素为 $(g, l)$，表示以 $i$ 为右端点、GCD 为 $g$ 的子数组的左端点起始位置为 $l$。
\end{itemize}

\subsection*{离线处理与查询排序}
\begin{enumerate}
    \item 将所有查询按右端点 $R$ 从小到大排序，并记录每个查询的原始索引，以便最后按输入顺序输出答案。
    \item 从左到右遍历数组，每次处理到位置 $i$ 时，同步处理所有 $R = i$ 的查询。
\end{enumerate}

\subsection*{维护以 $i$ 为右端点的 GCD 列表}
当遍历到位置 $i$ 时，按以下步骤维护当前的 GCD 列表：
\begin{enumerate}
    \item 初始化临时列表 $\mathrm{cur}$，先加入 $(a_i, i)$（对应子数组 $[i, i]$，其 GCD 为 $a_i$）。
    \item 取上一次处理 $i-1$ 时得到的列表 $\mathrm{prev}$，对其中每个元素 $(g, l)$，计算新的 GCD 值 $g' = \gcd(g, a_i)$，并将 $(g', l)$ 加入 $\mathrm{cur}$。
    \item 合并 $\mathrm{cur}$ 中相同的 GCD 值：对相同的 $g'$，仅保留最小的左端点 $l$（因为更小的 $l$ 对应覆盖范围更广的子数组）。合并后的列表作为新的 $\mathrm{prev}$，用于处理 $i+1$。
\end{enumerate}

\subsection*{利用树状数组维护不同 GCD 的计数}
为了快速查询区间 $[L, R]$ 内的不同 GCD 数量，我们使用树状数组维护标记信息：
\begin{itemize}
    \item 定义数组 $\mathrm{las}[g]$，表示 GCD 值 $g$ 最后一次被标记的左端点位置，初始为 0。
    \item 对于当前 $\mathrm{cur}$ 列表中的每个元素 $(g, l)$：
    \begin{enumerate}
        \item 若 $\mathrm{las}[g] \neq 0$，说明之前在位置 $\mathrm{las}[g]$ 标记过 $g$，需在树状数组中对 $\mathrm{las}[g]$ 位置执行单点减 1 操作（取消旧标记）。
        \item 在树状数组中对位置 $l$ 执行单点加 1 操作（标记新的最左位置）。
        \item 更新 $\mathrm{las}[g] = l$。
    \end{enumerate}
    \item 此时，树状数组中区间 $[L, R]$ 的和即为查询 $[L, R]$ 的答案：每个 GCD 值仅被标记一次，且仅当标记位置 $l \in [L, R]$ 时被计数。
\end{itemize}

\section*{复杂度分析}
\begin{itemize}
    \item \textbf{时间复杂度：}
    \begin{itemize}
        \item 维护 GCD 列表：每个元素 $a_i$ 对应的列表长度为 $O(\log a_i)$，总耗时 $O(N \log A)$（$A$ 为 $a_i$ 的最大值，约 $10^6$）。
        \item 树状数组操作：每次单点更新和区间求和均为 $O(\log N)$，总耗时 $O((N + Q) \log N)$。
        \item 综上，总时间复杂度为：
        \[
            O(N \log A + Q \log N)
        \]
    \end{itemize}
\end{itemize}

\subsection{曼哈顿距离与切比雪夫距离的相互转化}

\section*{曼哈顿距离转切比雪夫距离}
设原坐标系下两点坐标为 $(x_1, y_1)$ 和 $(x_2, y_2)$。定义变换：
\[
u = x + y,\qquad v = x - y
\]
则在新坐标系 $(u, v)$ 下，曼哈顿距离等于切比雪夫距离：
\[
|x_1 - x_2| + |y_1 - y_2| = \max(|u_1 - u_2|,\ |v_1 - v_2|)
\]
因此，处理最大曼哈顿距离问题时，只需在新坐标系下维护 $u$ 和 $v$ 的 $\min/\max$ 值，即可 $\mathcal{O}(1)$ 得到结果。

\section*{切比雪夫距离转曼哈顿距离}
设切比雪夫坐标系下两点坐标为 $(u_1, v_1)$ 和 $(u_2, v_2)$。定义逆变换：
\[
x = \frac{u + v}{2},\qquad y = \frac{u - v}{2}
\]
则原坐标系下的曼哈顿距离等于切比雪夫距离：
\[
\max(|u_1 - u_2|,\ |v_1 - v_2|) = |x_1 - x_2| + |y_1 - y_2|
\]
注意：若原坐标为整数，则 $u$ 与 $v$ 同奇偶，上式中的除法结果仍为整数或半整数，但距离公式依然成立。实际编程时可先全部乘以 $2$ 避免浮点数。

\subsection{判定可重集合是否相等}
\section*{定义}
\textbf{可重集合（Multiset）}：是指允许同一元素出现多次的集合。

\section*{问题描述}
给定数组 $a$，多次询问：判断区间 $[L_1,R_1]$ 与 $[L_2,R_2]$ 所对应的可重集合是否相等。设每次询问的两个可重集合分别为 $S$ 和 $T$。

\section*{朴素做法}
直接的做法包括：
\begin{itemize}
    \item 将两个区间内的元素排序逐一比较。
    \item 使用哈希表统计元素出现次数。
\end{itemize}
设 $|S|\le|T|$，单次询问时间复杂度为 $\mathcal{O}(|S|\cdot\log|S|)$。

\section*{随机化做法（随机赋权哈希）}
\subsection*{定理}
如果两个可重集合相等，那么两个集合的元素和相等。但仅靠元素和无法判定，反例：$S=\{2,4\},\ T=\{3,3\}$。

\subsection*{改进思路}
为每一种元素 $x$ 随机赋予一个权值 $w(x)$，定义集合的哈希值为：
\[
H(S)=\sum_{x\in S} w(x)
\]
判断两个集合是否相等，只需比较 $H(S)$ 和 $H(T)$。

\subsection*{正确性分析}
\begin{itemize}
    \item 若 $S=T$，则 $H(S)=H(T)$。
    \item 若 $S\neq T$，设 $cnt_S(x),cnt_T(x)$ 分别表示元素 $x$ 在集合中的出现次数，则
    \[
    \sum\bigl(cnt_S(x)-cnt_T(x)\bigr)\cdot w(x)=0
    \]
    这是一个关于随机变量 $w(x)$ 的线性组合。在 $w(x)$ 独立随机且值域足够大的情况下，上式成立的概率极低。
\end{itemize}

\subsection*{其他哈希形式}
也可以定义 $H(S)=\prod_{x\in S} w(x)$，同样可用于判定可重集合相等（本质仍为随机映射）。

\subsection*{区间查询实现}
\begin{itemize}
    \item 不带修：使用前缀和维护，单次查询 $\mathcal{O}(1)$。
    \item 待修：使用树状数组/线段树维护区间和，单次查询 $\mathcal{O}(\log n)$。
\end{itemize}

\section*{扩展问题}
给定数组 $a$，多次询问：判断区间 $[L_1,R_1]$ 中所有元素加上 $k$ 后，是否与区间 $[L_2,R_2]$ 的可重集合相等。

设常数 $B$，定义哈希函数：
\[
H(S)=\sum B^{a_i}
\]
则有：
\[
H([L,R])=\sum_{i=L}^{R} B^{a_i}
\]
区间整体加上 $k$ 后：
\[
H'([L,R])=\sum_{i=L}^{R} B^{a_i+k}=B^k\cdot\sum_{i=L}^{R} B^{a_i}=B^k\cdot H([L,R])
\]
因此只需判断 $B^k\cdot H([L_1,R_1])$ 与 $H([L_2,R_2])$ 的关系即可。
